## 4ï¸âƒ£ Document Review (â­ NEW FEATURE!)

### Understanding Document Review

**What Changed**: You can now approve/reject **individual documents** instead of waiting for all documents to be perfect.

**Benefits**:
- âœ… Faster processing
- âœ… Clear feedback to applicants
- âœ… Reduced back-and-forth
- âœ… Applicants know exactly what's wrong

### How to Review Documents

1. **Open Application** you're processing
2. **Navigate to Documents Tab**
3. **Click "Submitted" sub-tab**
4. You'll see list of uploaded documents with:
   - Document type
   - File name
   - Upload date
   - Current status (Pending / Approved / Rejected)

### Reviewing Individual Documents

For each document:

1. **Click the arrow icon** (â–¶ï¸) next to document name
2. **Document Review Modal Opens** showing:
   - Document preview (if image/PDF)
   - Document details
   - Action buttons

3. **Examine the Document**:

   **âœ… Good Document Checklist**:
   - Clear and readable
   - Complete (all pages present)
   - Correct document type
   - Valid/Not expired
   - Matches applicant information
   - Official/original (not obviously fake)

   **âŒ Reject If**:
   - Blurry or illegible
   - Wrong document uploaded
   - Expired (check dates!)
   - Missing pages
   - Appears altered/fake
   - Low quality scan

4. **Take Action**:

   **To Approve âœ…**:
   - Click **"Approve"** button
   - Document status changes to "Approved"
   - Green checkmark appears
   - Applicant receives notification: "Your [document name] has been approved"

   **To Reject âŒ**:
   - Click **"Reject"** button
   - **Rejection reason text area appears**
   - **IMPORTANT**: Provide specific, helpful reason:
  
   **Good Rejection Reasons**:
   ```
   "Image is too blurry to read the ID number. Please re-take photo in better lighting."
   
   "This appears to be a water bill, but we need electricity bill for proof of address."
   
   "Certificate has expired (valid until Dec 2025). Please provide current 2026 certificate."
   
   "Only page 1 is uploaded. Please combine all 3 pages into single PDF."
   ```
   
   **Bad Rejection Reasons** (Don't do this!):
   ```
   "Not clear" - Be specific about what's unclear
   "Wrong document" - Specify what's needed
   "Rejected" - Always explain why!
   ```

   - Click **"Confirm Rejection"**
   - Applicant receives notification with your reason
   - They can re-upload corrected document

### Document Review Best Practices

**âœ… DO**:
- Review documents same day they're uploaded
- Be specific in rejection reasons
- Help applicant understand what's needed
- Approve good documents immediately
- Check expiry dates carefully
- Zoom in if image is small

**âŒ DON'T**:
- Leave documents pending for days
- Use vague rejection reasons
- Reject without explaining why
- Be unnecessarily strict on minor issues
- Approve obviously fake documents

### Handling Re-Submissions

When applicant re-uploads after rejection:

1. Document shows as**"Pending"** again
2. You review it fresh
3. Check if they fixed the issue you mentioned
4. If fixed â†’ Approve âœ…

## 7ï¸âƒ£ Troubleshooting Common Issues

### Issue: "Can't find application in my queue"

**Solutions**:
1. Check filter settings - might be wrong status
2. Search by reference number
3. Check if assigned to another officer
4. Refresh page (F5)

### Issue: "Document preview not showing"

**Solutions**:
1. Check file format - only PDF/JPG/PNG supported
2. File might be corrupted - ask applicant to re-upload
3. Try downloading file instead
4. Clear browser cache

### Issue: "Can't change status"

**Solutions**:
1. Check required documents - all must be approved first
2. You might not have permission - contact admin
3. Application might be locked - check with manager

### Issue: "Applicant says they didn't receive notification"

**Solutions**:
1. Check their email in profile - might be wrong
2. Ask to check spam/junk folder
3. Add noreply@ncdc.gov.pg to contacts
4. Resend notification manually

### Issue: "System is slow"

**Solutions**:
1. Close unused browser tabs
2. Clear cache: Ctrl+Shift+Delete
3. Log out and log back in
4. Report to IT if persistent
